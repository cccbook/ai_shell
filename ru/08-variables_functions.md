# 8. Переменные и функции

---

## 8.1 Основы работы с переменными

### Базовое присваивание

```bash
# Строка
name="Алиса"
greeting="Привет, мир!"

# Число
age=25
count=0

# Пустое значение
empty=
empty2=""
```

### Чтение переменных

```bash
echo $name
echo ${name}    # Рекомендуемая форма, более явная

# В двойных кавычках
echo "Меня зовут ${name}"
```

### Распространённые ошибки

```bash
# Неправильно: пробелы вокруг =
name = "Алиса"   # Интерпретируется как команда

# Неправильно: без кавычек
greeting=Привет мир  # Выведет только "Привет"

# Правильно:
name="Алиса"
greeting="Привет мир"
```

## 8.2 Искусство кавычек

### Двойные кавычки `"`
Раскрывают переменные и подстановку команд

```bash
name="Алиса"
echo "Привет, $name"           # Привет, Алиса
echo "Сегодня $(date +%Y)"     # Сегодня 2026
```

### Одинарные кавычки `'`
Выводят буквально, ничего не раскрывают

```bash
name="Алиса"
echo 'Привет, $name'           # Привет, $name
echo 'Сегодня $(date +%Y)'     # Сегодня $(date +%Y)
```

### Без кавычек
Избегайте, если не уверены, что переменная не содержит пробелов

## 8.3 Специальные переменные

```bash
$0          # Имя скрипта
$1, $2...   # Позиционные параметры
$#          # Количество аргументов
$@          # Все аргументы (отдельные)
$*          # Все аргументы (как одна строка)
$?          # Код завершения последней команды
$$          # PID текущего процесса
$!          # PID последнего фонового процесса
$-          # Текущие опции оболочки
```

## 8.4 Массивы

### Базовое использование

```bash
# Определить массив
colors=("красный" "зелёный" "синий")

# Читать элементы
echo ${colors[0]}    # красный
echo ${colors[1]}    # зелёный

# Читать все
echo ${colors[@]}    # красный зелёный синий

# Длина массива
echo ${#colors[@]}   # 3
```

### Ассоциативные массивы (Bash 4+)

```bash
declare -A user
user["name"]="Алиса"
user["email"]="alice@example.com"

echo ${user["name"]}    # Алиса
```

## 8.5 Основы функций

### Определение функций

```bash
# Метод 1: ключевое слово function
function greet {
    echo "Привет!"
}

# Метод 2: прямое определение (рекомендуется)
greet() {
    echo "Привет!"
}
```

### Параметры функций

```bash
greet() {
    echo "Привет, $1!"
}

greet "Алиса"    # Привет, Алиса!
```

### Возвращаемые значения

```bash
# return: для кода завершения (0-255)
check() {
    if [[ $1 -gt 10 ]]; then
        return 0  # успех
    else
        return 1  # неудача
    fi
}

if check 15; then
    echo "Больше 10"
fi
```

## 8.6 Локальные переменные

```bash
counter() {
    local count=0
    ((count++))
    echo $count
}
```

## 8.7 Библиотеки функций

### Создание библиотеки

```bash
cat > lib.sh << 'EOF'
#!/bin/bash

log() {
    echo "[$(date +%H:%M:%S)] $@"
}

error() {
    echo "[$(date +%H:%M:%S)] ОШИБКА: $@" >&2
}

confirm() {
    read -p "$1 (д/н) " -n 1 -r
    echo
    [[ $REPLY =~ ^[Дд]$ ]]
}
EOF
```

### Использование библиотеки

```bash
#!/bin/bash

source lib.sh

log "Запуск процесса"
error "Что-то пошло не так"
confirm "Продолжить?" && echo "Продолжаем"
```

## 8.8 Краткий справочник

| Тема | Синтаксис | Описание |
|------|-----------|----------|
| Присваивание | `var=value` | Без пробелов вокруг = |
| Чтение | `$var` или `${var}` | Используйте `${var}` |
| Двойные кавычки | `"..."` | Раскрывают переменные |
| Одинарные кавычки | `'...'` | Без раскрытия |
| Аргументы | `$1`, `$2`, `$@` | Получить параметры |
| Функция | `name() { }` | рекомендуемое определение |
| Локальная var | `local var=value` | только в функции |
| Массив | `arr=(a b c)` | индексированные и ассоциативные |
| source | `source file.sh` | Загрузить библиотеку |
