# 16. Методы отладки

---

## 16.1 Подход ИИ к отладке

Когда люди сталкиваются с ошибками: паника, поиск в интернете, копирование
Когда ИИ сталкивается с ошибками: анализирует сообщение, определяет причину, выполняет исправление

Алгоритм отладки ИИ:
```
Анализ вывода ошибки → Понимание типа ошибки → Нахождение проблемы → Исправление → Проверка
```

---

## 16.2 `bash -x`: Трассировка выполнения

Самый простой способ отладки: добавьте флаг `-x`

```bash
bash -x script.sh
```

Выводит каждую выполненную строку с префиксом `+`:

```bash
+ mkdir -p test
+ cd test
+ echo 'Hello'
Hello
```

### Отладка только части скрипта

```bash
#!/bin/bash

echo "Это не отобразится"
set -x
# Отладка начинается здесь
name="Alice"
echo "Hello, $name"
set +x
# Отладка заканчивается здесь
echo "Это не отобразится"
```

## 16.3 Частые ошибки и их исправление

### Ошибка 1: Permission Denied (Отказано в доступе)

```bash
# Ошибка
./script.sh
# Вывод: Permission denied

# Исправление
chmod +x script.sh
./script.sh
```

### Ошибка 2: Command Not Found (Команда не найдена)

```bash
# Ошибка
python script.py
# Вывод: command not found: python

# Исправление: использовать полный путь
/usr/bin/python3 script.py
```

### Ошибка 3: Неопределённая переменная

```bash
#!/bin/bash
set -u

echo $undefined_var
# Вывод: bash: undefined_var: unbound variable

# Исправление: задать значение по умолчанию
echo ${undefined_var:-default}
```

## 16.4 Отладка с помощью `echo`

Когда `-x` недостаточно, добавьте `echo` вручную:

```bash
#!/bin/bash

echo "DEBUG: Вход в функцию"
echo "DEBUG: Параметр = $@"

process() {
    echo "DEBUG: Внутри process"
    local result=$(expensive_command)
    echo "DEBUG: result = $result"
}
```

## 16.5 Краткий справочник

| Команда | Описание |
|---------|----------|
| `bash -n script.sh` | Проверить только синтаксис |
| `bash -x script.sh` | Трассировка выполнения |
| `set -x` | Включить режим отладки |
| `set +x` | Выключить режим отладки |
| `trap 'echo cmd' DEBUG` | Трассировка каждой команды |

## 16.6 Упражнения

1. Выполните скрипт с `bash -x` и наблюдайте за форматом вывода
2. Используйте `set -x` в скрипте для отладки только определённой функции
3. Найдите команду, которая завершается ошибкой, проанализируйте сообщение и исправьте
4. Создайте корректную обработку ошибок с помощью `trap`
