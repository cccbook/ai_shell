# 4. Генерация и запись текста

---

## 4.1 Почему ИИ не нужен редактор

Рабочий процесс написания кода у большинства инженеров-людей:
1. Открыть редактор (VS Code, Vim, Emacs...)
2. Ввести код
3. Сохранить файл
4. Закрыть редактор

Для ИИ:
```
«Напиши программу на Python» = сгенерировать некоторый текст
«Сохрани эту программу в файл» = записать текст на диск
```

Процесс генерации кода ИИ — это **процесс генерации текста**. Поэтому ИИ использует текстовые инструменты Shell:

- `echo`: вывести одну строку текста
- `printf`: форматированный вывод
- `heredoc`: вывести многострочный текст (самое важное!)

---

## 4.2 `echo`: Простейший вывод

### Базовое использование

```bash
# Вывести строку
echo "Привет, мир!"

# Вывести переменную
name="Алиса"
echo "Привет, $name!"

# Вывести несколько значений
echo "Сегодня $(date +%Y-%m-%d)"
```

### Ловушки `echo`

```bash
# echo по умолчанию добавляет перенос строки
echo -n "Загрузка: "  # Без переноса строки
```

### Запись файлов с помощью `echo`

```bash
# Перезаписать
echo "Привет, мир!" > file.txt

# Добавить
echo "Вторая строка" >> file.txt
```

**Примечание**: использовать `echo` для многострочных файлов мучительно, поэтому ИИ почти никогда не использует его для кода. Звезда — это `heredoc`.

---

## 4.3 `printf`: Более мощный форматированный вывод

### Сравнение с `echo`

```bash
# printf поддерживает форматирование в стиле C
printf "Значение: %.2f\n" 3.14159
# Вывод: Значение: 3.14

printf "%s\t%s\n" "Имя" "Возраст"
```

### Создание таблиц

```bash
printf "%-15s %10s\n" "Имя" "Цена"
printf "%-15s %10.2f\n" "Айфон" 999.99
printf "%-15s %10.2f\n" "Макбук" 1999.99
```

---

## 4.4 heredoc: Основное оружие ИИ для написания кода

### Что такое heredoc?

heredoc — это специальный синтаксис Shell для **вывода многострочного текста дословно**.

```bash
cat << 'EOF'
Всё это содержимое
будет выведено дословно
включая переносы строк, пробелы
EOF
```

### Запись файлов (самое частое использование ИИ)

```bash
cat > program.py << 'EOF'
#!/usr/bin/env python3

def hello(name):
    print(f"Привет, {name}!")

if __name__ == "__main__":
    hello("Мир")
EOF
```

### Почему использовать `'EOF'` (одинарные кавычки)?

```bash
# Одинарные кавычки EOF: ничего не раскрывать
cat << 'EOF'
HOME это: $HOME
Сегодня: $(date)
EOF
# Вывод: HOME это: $HOME (не раскрыто)

# Двойные кавычки EOF или без кавычек: будет раскрыто
cat << EOF
HOME это: $HOME
EOF
# Вывод: HOME это: /home/ai (раскрыто)
```

**Выбор ИИ**: почти всегда использовать `'EOF'` (одинарные кавычки). Потому что коду обычно не нужно раскрытие переменных Shell.

## 4.5 ИИ пишет различные файлы с помощью heredoc

### Записать программу на Python

```bash
cat > src/main.py << 'EOF'
#!/usr/bin/env python3
"""Точка входа"""

import sys
import os

def main():
    print("Гибридный проект Python + C")
    print(f"Рабочий каталог: {os.getcwd()}")

if __name__ == "__main__":
    main()
EOF
```

### Записать Shell-скрипт

```bash
cat > scripts/deploy.sh << 'EOF'
#!/bin/bash
set -e

DEPLOY_HOST="server.example.com"

echo "🚀 Деплой на $DEPLOY_HOST"
rsync -avz --delete ./dist/ "$DEPLOY_HOST:/var/www/app/"
ssh "$DEPLOY_HOST" "systemctl restart myapp"
echo "✅ Деплой завершён!"
EOF

chmod +x scripts/deploy.sh
```

### Записать конфигурационный файл

```bash
cat > config.json << 'EOF'
{
    "site_name": "Мой блог",
    "author": "Аноним",
    "theme": "minimal"
}
EOF
```

### Записать Dockerfile

```bash
cat > Dockerfile << 'EOF'
FROM python:3.11-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY . .
CMD ["python", "-m", "http.server", "8000"]
EOF
```

## 4.6 Ловушки heredoc и решения

### Ловушка 1: Содержит одинарные кавычки

```bash
# Проблема: EOF в одинарных кавычках не позволяет использовать одинарные кавычки
cat << 'EOF'
He's going.
EOF
# Вывод: синтаксическая ошибка

# Решение: использовать двойные кавычки EOF
cat << "EOF"
He's going.
EOF
```

### Ловушка 2: Содержит `$` (но не нужно раскрытие)

```bash
# Проблема: двойные кавычки EOF раскрывают $
cat << "EOF"
Цена $100
EOF
# Вывод: Цена (пусто)

# Решение: экранировать отдельно
cat << "EOF"
Цена $$100
EOF
# Вывод: Цена $100
```

## 4.7 Практика: Создание полного проекта с нуля

```bash
# 1. Создать структуру каталогов
mkdir -p myblog/{src,themes,content}

# 2. Создать конфигурационный файл
cat > myblog/config.json << 'EOF'
{
    "site_name": "Мой блог",
    "author": "Аноним",
    "theme": "minimal"
}
EOF

# 3. Создать основную программу на Python
cat > myblog/src/blog.py << 'EOF'
#!/usr/bin/env python3
"""Простой генератор блога"""
import json
from pathlib import Path

def load_config():
    config_path = Path(__file__).parent.parent / "config.json"
    with open(config_path) as f:
        return json.load(f)

if __name__ == "__main__":
    config = load_config()
    print(f"Генерация: {config['site_name']}")
EOF

# 4. Создать скрипт сборки
cat > myblog/build.sh << 'EOF'
#!/bin/bash
set -e
echo "🔨 Сборка блога..."
cd "$(dirname "$0")"
python3 src/blog.py
echo "✅ Сборка завершена!"
EOF

chmod +x myblog/build.sh

# 5. Проверить структуру
find myblog -type f | sort
```

## 4.8 Краткая справка

| Инструмент | Назначение | Пример |
|------------|------------|--------|
| `echo` | Вывести одну строку | `echo "Привет"` |
| `echo -n` | Без переноса строки | `echo -n "Загрузка..."` |
| `printf` | Форматированный вывод | `printf "%s: %d\n" "возраст" 25` |
| `>` | Перезаписать файл | `echo "привет" > file.txt` |
| `>>` | Добавить в файл | `echo "привет" >> file.txt` |
| `<< 'EOF'` | heredoc (без раскрытия) | предпочтительно для кода |
| `<< "EOF"` | heredoc (с раскрытием) | редко используется |
