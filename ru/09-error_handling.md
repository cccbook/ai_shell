# 9. Обработка ошибок

---

## 9.1 Почему обработка ошибок важна

Без обработки ошибок:
- После отказа одной команды продолжаем с неверными операциями
- Можем удалить неправильные файлы
- Можем перезаписать важные данные
- Можем оставить систему в несогласованном состоянии

С обработкой ошибок:
- Останавливаемся сразу при ошибке
- Предоставляем понятные сообщения об ошибках
- Выполняем очистку перед выходом

---

## 9.2 Коды завершения

Каждая команда возвращает код завершения после выполнения:

- `0`: успех
- `не-ноль`: неудача

```bash
# Проверить код завершения последней команды
ls /tmp
echo $?  # Вывод: 0 (при успехе)

ls /nonexistent
echo $?  # Вывод: 2 (при неудаче)
```

## 9.3 `set -e`: Остановка при ошибке

```bash
#!/bin/bash
set -e

mkdir -p backup
cp important.txt backup/  # Если это не удастся, скрипт остановится здесь
rm important.txt          # Сюда не дойдём
```

### Когда использовать

Почти все скрипты должны использовать `set -e`:
- Скрипты инициализации
- Скрипты деплоя
- Скрипты автоматического тестирования

---

## 9.4 `set -u`: Ошибка при неопределённых переменных

```bash
#!/bin/bash
set -u

echo $undefined_var
# Вывод: bash: undefined_var: unbound variable
```

### Комбинированное использование

```bash
#!/bin/bash
set -euo pipefail

# -e: остановка при ошибке
# -u: ошибка при неопределённой переменной
# -o pipefail: конвейер завершается неудачей, если любая команда не удалась
```

**Это стандартный заголовок скрипта AI!**

## 9.5 `trap`: Корректная обработка ошибок

### Базовое использование

```bash
#!/bin/bash
set -euo pipefail

cleanup() {
    echo "Очистка..."
    rm -f /tmp/tempfile
}

trap cleanup EXIT

# Основная программа
echo "Запуск процесса..."
```

### Перехват ошибок

```bash
#!/bin/bash
set -euo pipefail

error_handler() {
    local exit_code=$?
    echo "❌ Скрипт завершился неудачей в строке $1, код завершения: $exit_code"
    exit $exit_code
}

trap 'error_handler $LINENO' ERR
```

## 9.6 Пользовательские функции для ошибок

```bash
#!/bin/bash
set -euo pipefail

die() {
    echo "❌ $@" >&2
    exit 1
}

warn() {
    echo "⚠️ $@"
}

need_command() {
    command -v "$1" &>/dev/null || die "Требуется команда: $1"
}

need_file() {
    [[ -f "$1" ]] || die "Требуется файл: $1"
}
```

## 9.7 Краткий справочник

| Команда | Описание |
|---------|----------|
| `$?` | Код завершения последней команды |
| `set -e` | Остановка при ошибке |
| `set -u` | Ошибка при неопределённой переменной |
| `set -o pipefail` | Конвейер неудачен, если любая команда неудачна |
| `set -euo pipefail` | Комбинация (рекомендуется) |
| `trap 'func' EXIT` | Выполнить при выходе |
| `trap 'func' ERR` | Выполнить при ошибке |
| `trap 'func' INT` | Выполнить при Ctrl+C |
| `exit 1` | Выйти с кодом ошибки 1 |

---

## 9.8 Упражнения

1. Напишите скрипт с `set -euo pipefail`, который выводит "Произошла ошибка" при неудаче
2. Создайте функцию `die()`, которая выводит сообщение и завершает работу
3. Используйте `trap` для отображения "До свидания" при выходе из скрипта
4. Напишите скрипт деплоя с автоматическим откатом при неудаче
