# 19. Режим сотрудничества ИИ + Shell

---

## 19.1 Новая форма взаимодействия человека и машины

Традиционное программирование:
- Люди вводят текст с клавиатуры
- Люди используют мышь для работы с IDE
- Люди выполняют и тестируют

Программирование в эпоху ИИ:
- Люди описывают требования
- ИИ генерирует Shell-команды и скрипты
- Люди проверяют и выполняют
- Люди и ИИ отлаживают вместе

---

## 19.2 Описание требований ИИ

### Хорошие описания

```bash
# Чёткая задача
"Find all .log files larger than 100MB in /var/log"

# Включить ожидаемый формат вывода
"List all .py files, showing: line_count filename per line"

# Указать ограничения
"Compress all .txt files, but skip any containing 'test'"
```

### Плохие описания

```bash
# Слишком расплывчато
"help me process logs"

# Нереально
"help me write an operating system"
```

## 19.3 Паттерны Shell-команд, генерируемых ИИ

### Паттерн 1: Одиночная команда

```bash
# Человек спрашивает: найти 10 Python-файлов с наибольшим количеством строк
find . -name "*.py" -exec wc -l {} + | sort -rn | head -10
```

### Паттерн 2: Shell-скрипт

```bash
# Человек спрашивает: пакетная обработка изображений
cat > process_images.sh << 'EOF'
#!/bin/bash
for img in *.jpg; do
    convert "$img" -resize 800x600 "thumb_$img"
done
EOF
```

## 19.5 Позвольте ИИ помочь с отладкой

```bash
# Человек: получил ошибку после запуска этой команды
$ ./deploy.sh
# Вывод: /bin/bash^M: bad interpreter: No such file or directory

# ИИ: это проблема с окончаниями строк Windows. Выполните:
sed -i 's/\r$//' deploy.sh
```
