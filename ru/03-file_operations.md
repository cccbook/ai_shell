# 3. Операции с файлами

---

## 3.1 Модель файловой системы в сознании ИИ

Прежде чем углубляться в отдельные команды, поймите, как ИИ видит файловую систему.

Инженеры-люди обычно видят файловую систему **визуально** — как Проводник Windows или Finder macOS, понимая через иконки и формы папок.

Видение ИИ полностью отличается:

```
path          = абсолютное расположение /home/user/project/src/main.py
relative      = идти вниз от текущего места
nodes         = каждый файл или каталог — это «узел»
attributes    = права, размер, временные метки, владелец
type          = обычный файл(-), каталог(d), ссылка(l), устройство(b/c)
```

Когда ИИ выполняет `ls -la`, он видит:

```
drwxr-xr-x  5 ai  staff  170 Mar 22 10:30 .
drwxr-xr-x  3 ai  staff  102 Mar 22 09:00 ..
-rw-r--r--  1 ai  staff  4096 Mar 22 10:30 .env
-rw-r--r--  1 ai  staff  8192 Mar 22 10:31 README.md
drwxr-xr-x  3 ai  staff   96 Mar 22 10:30 src
drwxr-xr-x  2 ai  staff   64 Mar 22 10:30 tests
-rw-r--r--  1 ai  staff  2048 Mar 22 10:32 package.json
```

ИИ может немедленно прочитать:
- Какие являются каталогами (`d`)
- Какие являются скрытыми файлами (начинаются с `.`)
- У кого какие права
- Размеры файлов (определить большие файлы)
- Время последнего изменения

## 3.2 `ls`: Самая используемая команда ИИ

Практически перед каждой операцией ИИ выполняет `ls` для подтверждения текущего состояния.

### Частые комбинации `ls` у ИИ

```bash
# Базовый список
ls

# Показать скрытые файлы (очень важно!)
ls -a

# Длинный формат (подробная информация)
ls -l

# Длинный формат + скрытые файлы (самый частый)
ls -la

# Сортировка по времени изменения (новые первые)
ls -lt

# Сортировка по времени изменения (старые первые)
ls -ltr

# Размеры в человекочитаемом формате (K, M, G)
ls -lh

# Показать только каталоги
ls -d */

# Рекурсивно показать все файлы
ls -R

# Показать номера inode (полезно для жёстких ссылок)
ls -li
```

### Реальный рабочий процесс ИИ

```bash
cd ~/project && ls -la

# Результат:
# drwxr-xr-x  5 ai  staff  170 Mar 22 10:30 .
# drwxr-xr-x  3 ai  staff  102 Mar 22 09:00 ..
# -rw-r--r--  1 ai  staff  4096 Mar 22 10:30 .env
# -rw-r--r--  1 ai  staff  8192 Mar 22 10:31 README.md
# drwxr-xr-x  3 ai  staff   96 Mar 22 10:30 src
# drwxr-xr-x  2 ai  staff   64 Mar 22 10:30 tests
# -rw-r--r--  1 ai  staff  2048 Mar 22 10:32 package.json

# Анализ ИИ: есть .env файл, каталоги src и tests, package.json
# Это проект Node.js
```

## 3.3 `cd`: Каталог, который ИИ никогда не забывает

### Привычки ИИ с `cd`

```bash
# Перейти в домашний каталог
cd ~

# Перейти в предыдущий каталог (очень полезно!)
cd -

# Перейти в родительский каталог
cd ..

# Перейти в подкаталог
cd src

# Перемещение по глубоким путям (заслуга Tab-автодополнения)
cd ~/project/backend/api/v2/routes
```

### Паттерн ИИ: `cd` + `&&`

Это один из самых распространённых паттернов ИИ:

```bash
# Сначала cd, только после подтверждения успеха выполнить следующую команду
cd ~/project && ls -la
```

### Частые ошибки

```bash
# Ошибка: не убедились, что каталог существует
cd nonexistent
# Вывод: bash: cd: nonexistent: No such file or directory

# Подход ИИ: сначала проверить
[ -d "nonexistent" ] && cd nonexistent || echo "Каталог не существует"
```

## 3.4 `mkdir`: Искусство создания каталогов

### Базовое использование

```bash
# Создать один каталог
mkdir myproject

# Создать несколько каталогов
mkdir src tests docs

# Создать вложенные каталоги (-p — ключевой!)
mkdir -p project/src/components project/tests
```

### Почему ИИ почти всегда использует `-p`

Флаг `-p` (parents) означает:
1. Если каталог существует, **ошибки не будет**
2. Если родительского каталога нет, **создать автоматически**

### Типичный паттерн создания проекта у ИИ

```bash
# Создать стандартную структуру проекта
mkdir -p project/{src,tests,docs,scripts,config}
```

## 3.5 `rm`: Искусство удаления

**Предупреждение**: это одна из самых опасных команд в Shell.

### Базовое использование

```bash
# Удалить файл
rm file.txt

# Удалить каталог (нужен -r)
rm -r directory/

# Удалить каталог и всё внутри (опасно!)
rm -rf directory/
```

### Опасность `rm -rf`

```bash
# Никогда не выполняйте это как root!
# rm -rf /

# Если случайно добавить лишний пробел:
rm -rf * 
# (пробел) = rm -rf удалит всё в текущем каталоге
```

## 3.6 `cp`: Копирование файлов и каталогов

### Базовое использование

```bash
# Копировать файл
cp source.txt destination.txt

# Копировать каталог (нужен -r)
cp -r source_directory/ destination_directory/

# Показать прогресс при копировании (-v verbose)
cp -v large_file.iso /backup/

# Интерактивный режим (спросить перед перезаписью)
cp -i *.py src/
```

### Сила подстановочных знаков

```bash
# Копировать все .txt файлы
cp *.txt backup/

# Копировать все файлы изображений
cp *.{jpg,png,gif,webp} images/
```

## 3.7 `mv`: Перемещение и переименование

### Базовое использование

```bash
# Переместить файл
mv file.txt backup/

# Переместить и переименовать
mv oldname.txt newname.txt

# Пакетное переименование
for f in *.txt; do
    mv "$f" "${f%.txt}.md"
done
```

## 3.8 Краткая справка

| Команда | Базовое использование | Частые флаги | Заметка ИИ |
|---------|----------------------|--------------|------------|
| `ls` | `ls [path]` | `-l` длинный, `-a` скрытые, `-h` человек | `ls -la` всегда хорош |
| `cd` | `cd [path]` | `-` предыдущий, `..` родитель | `cd xxx &&` — хорошая привычка |
| `mkdir` | `mkdir [dir]` | `-p` вложенные | почти всегда использовать `-p` |
| `rm` | `rm [file]` | `-r` рекурсивно, `-f` принудительно | берегитесь `rm -rf /*` |
| `cp` | `cp [src] [dst]` | `-r` каталог, `-i` спросить, `-p` сохранить | используйте `-i` для безопасности |
| `mv` | `mv [src] [dst]` | `-i` спросить, `-n` не перезаписывать | это переименование |

## 3.9 Упражнения

1. Используйте `mkdir -p` для создания трёхуровневой вложенной структуры каталогов, затем подтвердите с помощью `tree` или `find`
2. Скопируйте большой файл с `cp -v` и посмотрите вывод
3. Пакетно переименуйте 10 файлов `.txt` в `.md` с помощью `mv`
4. Удалите тестовый файл с `rm -i`, чтобы испытать запрос подтверждения
