# 17. Оптимизация производительности

---

## 17.1 Почему производительность Shell важна

Shell-скрипты часто используются для:
- Пакетной обработки большого количества файлов
- Задач автоматизации
- CI/CD пайплайнов

Медленный скрипт может задержать весь процесс на часы. Оптимизация Shell-скриптов может значительно повысить эффективность.

---

## 17.2 Избегайте внешних команд

```bash
# Медленно: внешние команды
for f in *.txt; do
    name=$(basename "$f")
    echo "$name"
done

# Быстро: встроенные функции Shell
for f in *.txt; do
    echo "${f##*/}"
done
```

### Встроенные команды vs Внешние команды

| Медленно | Быстро | Описание |
|----------|--------|----------|
| `$(cat file)` | `$(<file)` | Прямое чтение |
| `$(basename $f)` | `${f##*/}` | Расширение параметров |
| `$(expr $a + $b)` | `$((a + b))` | Арифметика |
| `$(echo $var)` | `"$var"` | Прямое использование |

---

## 17.3 Используйте `while read` вместо `for`

```bash
# Медленно: for + подстановка команд
for line in $(cat file.txt); do
    process "$line"
done

# Быстро: while read
while IFS= read -r line; do
    process "$line"
done < file.txt
```

## 17.4 Параллельная обработка

### Используйте `&` и `wait`

```bash
#!/bin/bash

task1 &
task2 &
task3 &

wait

echo "All tasks complete"
```

### Используйте `xargs -P`

```bash
# Последовательно
cat files.txt | xargs -I {} process {}

# Параллельно (4 одновременных процесса)
cat files.txt | xargs -P 4 -I {} process {}
```

---

## 17.5 Избегайте подпроцессов

```bash
# Медленно: подпроцесс в каждой итерации
for f in *.txt; do
    content=$(cat "$f")
    echo "$content" | wc -l
done

# Быстро: один подпроцесс
while IFS= read -r f; do
    wc -l "$f"
done < <(ls *.txt)
```

---

## 17.6 Краткий справочник

| Оптимизация | Медленно | Быстро |
|-------------|----------|--------|
| Чтение файла | `$(cat file)` | `$(<file)` или `while read` |
| Путь | `$(basename $f)` | `${f##*/}` |
| Математика | `$(expr $a + $b)` | `$((a + b))` |
| Параллелизм | последовательно | `&` + `wait` или `xargs -P` |
